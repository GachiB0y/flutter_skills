# Data Models / Модели данных

> Этот файл -- справочник. Основной скилл: feature-architecture/SKILL.md

## Обзор

В проекте используй **plain Dart classes** (без freezed/json_serializable) для моделей данных.

Одна модель используется на всех уровнях (Data -> BLoC -> UI) для простоты и избежания избыточных преобразований.

## 1. Типы моделей

### 1.1. Universal Entity (Универсальная сущность)

Единая модель для API, бизнес-логики и UI.

Характеристики:

- Суффикс: `*Entity` или без суффикса
- Расположение: `data/models/` или корень фичи
- Сериализация: `fromJson()` / `toJson()` с `Map<String, Object?>`
- Иммутабельность: `@immutable`
- Методы: `copyWith()`, `==`, `hashCode`, `toString()`
- Опционально: `Comparable<T>` для сортировки
- Используется: Data Source -> Repository -> BLoC -> UI

### 1.2. Database Wrapper (Обёртка для БД)

Обёртка над моделью из базы данных для использования в остальном коде.

Используй когда:

- Модель из БД (Drift) содержит служебную информацию
- Нужно объединить несколько таблиц в одну сущность
- БД модель не подходит для BLoC/UI

Характеристики:

- Суффикс: `*WithRelations` или `With*`
- Расположение: `database/models/` (создается) -> `data/models/` (используется)
- Методы: `fromDatabase()` в основной модели для преобразования
- Композиция: содержит Drift модели + дополнительные данные

### 1.3. Value Object (Объект-значение)

Неизменяемые данные без идентификатора, для передачи параметров.

Характеристики:

- Без суффикса или `*Data`
- Расположение: `data/models/` или корень фичи
- Все поля `final` + `const` конструктор
- Методы: `==`, `hashCode` (без `copyWith`, `toString`)
- Без сериализации (создается в коде)

## 2. Выбор типа модели

| Сценарий | Тип модели | Пример |
|---|---|---|
| API -> Repository -> BLoC -> UI | Universal Entity | `UserEntity` везде |
| База данных -> основной код | Database Wrapper -> Universal Entity | `HistoryEntry.fromDatabase()` |
| Форма с валидацией -> API | Form Data -> Universal Entity | `RegistrationData.toEntity()` |
| Параметры функции/изолят | Value Object | `AnalyzeImageData` |

## 3. Ключевые принципы

1. **Одна модель на всех уровнях.** Universal Entity используется в Data -> BLoC -> UI без преобразований
2. **Database Wrapper только для БД.** Преобразуй в Universal Entity через `fromDatabase()`
3. **Иммутабельность.** Все поля `final`, конструкторы `const` где возможно
4. **Type Safety.** Используй `Object?` для `fromJson()`, делай явное приведение типов
5. **Отдельные классы для форм.** Form Data с валидацией, преобразуй в Universal Entity
6. **Без избыточных преобразований.** Нет DTO <-> Entity на каждом уровне
7. **Без кодогенерации.** Plain Dart classes для полного контроля
